## Known bugs, blockers, risks

### Active bug being discussed (2026-05-11)

Client tested login on `9jaRidePro-Customer-Universal.apk` and hit two errors in sequence:

1. `An internal error has occurred. [ BILLING_NOT_ENABLED ]` when entering a real Nigerian phone number `+234 8100238560`.
2. `We have blocked all requests from this device due to unusual activity. Try again later.` after several retries of error #1.

Diagnosis: both are Firebase server-side, not app bugs.

- BILLING_NOT_ENABLED happens because `jaride-pro` is on the Spark (free) plan. Firebase discontinued free SMS phone-auth for new projects in late 2023. Free tier now only sends OTP to pre-registered test numbers.
- The "blocked" error is Firebase's anti-abuse throttle, triggered by repeated failed attempts. It clears in a few hours to ~24 hours. Each retry extends the block, so the client should stop tapping Next.

Three resolution options presented to the user:

- A. Upgrade Firebase to Blaze plan. Fastest. About $0.05 per SMS to Nigeria. No code change.
- B. Integrate Termii instead. Cheaper for Nigeria (~N4 per SMS) but needs Termii API keys from client and backend work to swap providers.
- C. Use test phone numbers `+234 1234567890` / `+234 0987654321` (both OTP `123456`) for the client's own evaluation while deciding A vs B.

A draft reply message was prepared for the client. The user has not yet sent it or decided which path to push. See "Next best action" below.
